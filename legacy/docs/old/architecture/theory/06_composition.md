# Композиция сущностей в ECS

В ProjectV проектирование объектов базируется на принципе композиции компонентов вместо иерархического наследования.
Свойства и поведение сущности определяются набором ассоциированных с ней данных (компонентов).

## 1. Теги (Tags)

Теги представляют собой компоненты без данных (пустые структуры), используемые для маркировки сущностей и фильтрации в
системах.

```cpp
// Определения тегов
struct Enemy {};
struct Player {};
struct Visible {};

// Использование в Flecs:
world.entity("Orc")
    .add<Enemy>()
    .add<Visible>();
```

Использование тегов позволяет динамически изменять поведение сущности во время выполнения. Например, удаление тега
`Visible` исключает сущность из обработки системой рендеринга без влияния на другие системы (физика, ИИ).

## 2. Префабы (Prefabs)

Префабы являются шаблонами сущностей, которые хранят предопределенный набор компонентов и их значений. Они используются
для эффективного создания множества однотипных объектов.

```cpp
// Создание префаба
auto GoblinPrefab = world.prefab("Goblin")
    .set<Health>({50, 50})
    .set<Speed>({2.0f})
    .add<Enemy>();

// Инстанцирование сущностей на основе префаба
auto g1 = world.entity().is_a(GoblinPrefab).set<Position>({10, 0, 5});
auto g2 = world.entity().is_a(GoblinPrefab).set<Position>({20, 0, 10});
```

Изменение значений компонентов в префабе автоматически отражается на всех инстанцированных сущностях, что упрощает
настройку баланса и характеристик объектов.

## 3. Отношения (Relationships)

Отношения (Pairs) позволяют устанавливать типизированные связи между сущностями без хранения прямых указателей в
компонентах.

```cpp
// Установка связи (Отношение, Субъект)
sword.add<EquippedBy>(warrior); // Меч экипирован воином
chest.add<Contains>(gold);      // Сундук содержит золото
```

Механизм отношений позволяет выполнять сложные запросы к миру, например, поиск всех предметов, принадлежащих конкретному
персонажу, обеспечивая прозрачность связей в архитектуре.

## 4. Формирование поведения через компоненты

Композиция позволяет создавать объекты с уникальными свойствами путем комбинирования стандартных блоков данных.

| Тип сущности | Health | Position | Speed | PlayerTag | EnemyTag | FlyingTag |
|:-------------|:------:|:--------:|:-----:|:---------:|:--------:|:---------:|
| **Герой**    |  [x]   |   [x]    |  [x]  |    [x]    |   [ ]    |    [ ]    |
| **Стена**    |  [x]   |   [x]    |  [ ]  |    [ ]    |   [ ]    |    [ ]    |
| **Дракон**   |  [x]   |   [x]    |  [x]  |    [ ]    |   [x]    |    [x]    |

Добавление компонента (например, `FlyingTag`) мгновенно включает сущность в выборку соответствующих систем обработки,
обеспечивая высокую модульность кода. Принципы взаимодействия систем описаны
в [System Communication](03_system-communication.md).
