## Основные понятия

### Tracy

**Кроссплатформенный профилировщик реального времени** для CPU и GPU. Позволяет измерять производительность кода,
выявлять узкие места и оптимизировать приложения.

### Профилирование (Profiling)

**Процесс измерения производительности** кода для выявления узких мест (bottlenecks), оптимизации и улучшения
эффективности.

### Zone (Зона)

**Блок кода, окружённый макросами профилирования**. Зоны измеряют время выполнения и отображаются в timeline Tracy как
цветные блоки.

**Пример:**

```cpp
{
    ZoneScopedN("MyFunction");
    // Код для профилирования...
}
```

### Frame (Кадр)

**Один цикл выполнения** игрового или рендеринг цикла. Отмечается макросом `FrameMark` для измерения FPS и времени
кадра.

### Plot (График)

**Визуализация числовых значений** во времени. Используется для отслеживания метрик (FPS, использование памяти,
количество объектов).

### Message (Сообщение)

**Текстовое сообщение**, привязанное ко времени в timeline. Используется для логирования событий и отладочной
информации.

### Context (Контекст)

**Окружение профилирования** для конкретного API (Vulkan, OpenGL, DirectX). Например, `VkCtx` для Vulkan.

## Типы данных

### Timeline (Таймлайн)

**Визуальное представление** выполнения кода во времени. Показывает зоны, кадры, сообщения и другие события в
хронологическом порядке.

### Call Stack (Стек вызовов)

**Цепочка вызовов функций**, приведшая к текущей точке выполнения. Используется для отладки и понимания flow выполнения.

### GPU Zone (GPU зона)

**Зона профилирования** для операций GPU (шейдеры, compute dispatch, memory transfers). Показывает время выполнения на
GPU.

### Memory Allocation (Аллокация памяти)

**Выделение блока памяти**, отслеживаемое Tracy. Показывает размер, расположение и время жизни выделенной памяти.

### Thread (Поток)

**Отдельная последовательность выполнения**. Tracy отслеживает выполнение каждого потока отдельно и показывает их
параллельно в timeline.

### Fiber (Файбер)

**Легковесный поток** пользовательского уровня. Tracy поддерживает профилирование fibers для асинхронных операций.

## Инструменты профилирования

### ZoneScoped

**Макрос для создания зоны** с именем текущей функции. Автоматически определяет имя функции.

### ZoneScopedN

**Макрос для создания именованной зоны**. Позволяет задать произвольное имя для зоны.

### FrameMark

**Макрос для отметки кадра**. Используется в игровом цикле для измерения времени кадра и FPS.

### TracyPlot

**Макрос для добавления точки** на график. Отслеживает числовые значения во времени.

### TracyMessage

**Макрос для отправки сообщения**. Добавляет текстовое сообщение в timeline.

### TracyAlloc / TracyFree

**Макросы для отслеживания** аллокаций и освобождений памяти.

### TracyVkZone

**Макрос для создания зоны** Vulkan. Профилирует выполнение команд GPU.

## Архитектурные термины

### Клиент (Client)

**Часть Tracy, встроенная в приложение**. Собирает данные профилирования и отправляет их на сервер.

### Сервер (Server)

**Часть Tracy, принимающая данные**. Агрегирует данные от клиентов и предоставляет Web UI.

### Web UI

**Веб-интерфейс** для визуализации профилировочных данных. Доступен через браузер по адресу `http://localhost:8080`.

### Transport (Транспорт)

**Механизм передачи данных** от клиента к серверу. По умолчанию использует TCP порт 8947.

### Overhead (Накладные расходы)

**Дополнительные затраты ресурсов** (CPU, память) на сбор данных профилирования. Tracy минимизирует overhead (<1% CPU).

### Sampling (Сэмплирование)

**Периодический сбор данных** с определённой частотой. Используется для снижения overhead.

### Real-time (Реальное время)

**Немедленная визуализация** данных профилирования без остановки приложения.

## Метрики и измерения

### FPS (Frames Per Second)

**Количество кадров в секунду**. Основная метрика производительности игр и графических приложений.

### Frame Time (Время кадра)

**Время выполнения одного кадра** в миллисекундах. Целевое значение для 60 FPS: <16.6 мс.

### CPU Time (Время CPU)

**Время выполнения** на процессоре. Измеряется для отдельных зон и функций.

### GPU Time (Время GPU)

**Время выполнения** на графическом процессоре. Измеряется для шейдеров, compute операций и memory transfers.

### Memory Usage (Использование памяти)

**Количество используемой памяти** в байтах, килобайтах, мегабайтах.

### Allocation Rate (Частота аллокаций)

**Количество аллокаций памяти** в единицу времени. Высокий rate может указывать на проблемы.

### Call Stack Depth (Глубина стека вызовов)

**Количество фреймов** в цепочке вызовов. Используется для отладки сложных вызовов.

### Latency (Задержка)

**Время между событием** и его обработкой. Например, input latency (задержка ввода).

## Интеграционные термины

### CMake Integration

**Интеграция Tracy** в систему сборки CMake. Добавление подпроекта, определение макросов и линковка библиотек.

### Vulkan Integration

**Интеграция Tracy** с Vulkan API. Создание контекста Vulkan, профилирование командных буферов и шейдеров.

### ECS Integration

**Интеграция Tracy** с Entity Component System (flecs). Профилирование систем, запросов и итераций.

### SDL Integration

**Интеграция Tracy** с Simple DirectMedia Layer. Профилирование событий, времени кадра и window management.

### Memory Allocator Integration

**Интеграция Tracy** с аллокаторами памяти (VMA, custom allocators). Отслеживание аллокаций и освобождений.

### Multi-threading Integration

**Интеграция Tracy** с многопоточным кодом. Именование потоков, профилирование критических секций.

## Частые вопросы

### Чем отличается Zone от Frame?

- **Zone** — произвольный блок кода любой длительности
- **Frame** — один цикл выполнения приложения (обычно 16-33 мс)

### Что такое overhead Tracy?

Дополнительные затраты CPU и памяти на сбор данных профилирования. Tracy имеет очень низкий overhead (<1% CPU).

### Какой порт использует Tracy?

По умолчанию TCP порт **8947** для клиент-серверного соединения и порт **8080** для Web UI.

### Нужен ли интернет для Tracy?

Нет, Tracy работает локально. Сервер и клиент могут быть на одной машине.

### Поддерживает ли Tracy многопоточность?

Да, Tracy полностью поддерживает многопоточность, именование потоков и анализ contention.

### Как Tracy интегрируется с Vulkan?

Через заголовочные файлы `TracyVulkan.hpp` и макросы `TracyVkZone`, `TracyVkCollect`.

### Можно ли использовать Tracy в релизных сборках?

Да, но рекомендуется уменьшить уровень детализации (отключить callstack, использовать `TRACY_ONLY_FRAME`).
