# ECS: Основные концепции

Entity Component System (ECS) — это архитектура, разделяющая данные, поведение и идентичность объектов. В ProjectV мы
используем библиотеку **Flecs**.

## 1. Сущности (Entities)

Сущность — это просто **ID** (`uint64_t`). Это не объект в памяти, не класс, не структура.
Это просто число, которое говорит "Я существую".

* **Пример:** `Entity 42` (Игрок). `Entity 105` (Меч).
* **Свойство:** К сущности можно прикрепить (или открепить) компоненты в любой момент.

## 2. Компоненты (Components)

Компоненты — это **Данные** (`struct`).
Они не имеют методов (кроме конструкторов и простых геттеров).
Они не знают о других компонентах.

### Правила ProjectV:

* **POD-типы (Plain Old Data):** `float`, `int`, `vec3`.
* **Минимум `std::string`:** Строки — это аллокации. Используйте `const char*` или строковые ID, если возможно.
* **Нет логики:** В компоненте `Position` не должно быть метода `move()`.

```cpp
struct Position { float x, y, z; };
struct Velocity { float x, y, z; };
struct Name { const char* value; };
```

## 3. Системы (Systems)

Системы — это **Логика** (Функции).
Они обрабатывают сущности, у которых есть *определенный набор* компонентов.

### Query-Based Logic

Система говорит миру: *"Дай мне все сущности, у которых есть `Position` И `Velocity`"*.
Мир (World) дает ей итератор. Система бежит по итератору и обновляет данные.

```cpp
// Пример (псевдокод)
void MoveSystem(iter& it) {
    auto p = it.term<Position>(1);
    auto v = it.term<Velocity>(2);

    for (auto i : it) {
        p[i].x += v[i].x * dt;
        p[i].y += v[i].y * dt;
        p[i].z += v[i].z * dt;
    }
}
```

## 4. Мир (World)

Мир — это **База Данных**.
Он хранит все сущности, все компоненты и все системы.
Он управляет памятью (выделяет, освобождает).
Он запускает системы (Pipeline).

### Порядок выполнения

В ECS порядок систем важен.

1. **InputSystem**: Читает ввод.
2. **PhysicsSystem**: Обновляет позиции.
3. **CameraSystem**: Обновляет матрицу вида.
4. **RenderSystem**: Рисует мир.

## Преимущества для Вокселей

1. **Масштабируемость:** Можно добавить 100 000 частиц. Система `ParticleSystem` просто обработает их в цикле. Никаких
   `virtual update()` для каждой частицы.
2. **Кэш-локальность:** Данные частиц лежат массивом. Процессор счастлив.
3. **Гибкость:** Хотите, чтобы воксель падал? Добавьте ему компонент `Gravity`. Не надо наследовать `FallingVoxel` от
   `Voxel`.
