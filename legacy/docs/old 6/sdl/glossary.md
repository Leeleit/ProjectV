## Базовые понятия C/SDL

Понятия, полезные при чтении документации SDL:

| Термин                     | Пояснение                                                                                                                                                                                                                                                                                                                                       |
|----------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Handle**                 | В SDL — обычно указатель на неполный тип (например, `SDL_Window*`). Не разыменовывается; функции SDL принимают его и работают с внутренними данными.                                                                                                                                                                                            |
| **Union SDL_Event**        | В C union — в каждый момент активно одно из полей. У `SDL_Event` поле `type` определяет, какое поле (`event.key`, `event.window`, `event.quit`) содержит данные. После проверки `type` обращайтесь к соответствующему полю. Данные события **не сохраняются** после возврата из callback — нельзя хранить указатель на `event->key.text` и т.п. |
| **Callback**               | Функция, которую вы реализуете, а SDL вызывает при наступлении события. По сути — указатель на функцию (аналог `std::function`).                                                                                                                                                                                                                |
| **Uint32, Sint32, Uint64** | Целочисленные типы SDL (`Uint32` ≈ `uint32_t`, `Sint32` ≈ `int32_t`). Используются для единообразия на разных платформах.                                                                                                                                                                                                                       |
| **bool**                   | В SDL функции возвращают `true`/`false`; `true` обычно означает успех.                                                                                                                                                                                                                                                                          |

**SDL.h и SDL_vulkan.h:** `SDL.h` **не** включает `SDL_vulkan.h`. Для Vulkan-функций нужен отдельный
`#include "SDL3/SDL_vulkan.h"`.

## Инициализация

| Термин                 | Объяснение                                                                                                                                            |
|------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------|
| **SDL_Init**           | Инициализирует подсистемы SDL (video, audio, input и т.д.). Возвращает `true` при успехе. Вызывать в начале приложения, желательно на main thread.    |
| **SDL_Quit**           | Деинициализирует все подсистемы. Вызывать перед выходом. При `SDL_MAIN_USE_CALLBACKS` SDL вызывает сам после `SDL_AppQuit`.                           |
| **SDL_InitFlags**      | Флаги для `SDL_Init`: `SDL_INIT_VIDEO`, `SDL_INIT_AUDIO`, `SDL_INIT_JOYSTICK` и др. Для игры с Vulkan обычно достаточно `SDL_INIT_VIDEO`.             |
| **SDL_INIT_VIDEO**     | Подсистема видеовывода и окон. **Подразумевает** `SDL_INIT_EVENTS`. Включает базовую поддержку ввода (клавиатура, мышь). Обязательна для Vulkan-окна. |
| **SDL_SetAppMetadata** | Опционально: заполняет метаданные приложения (имя, версия). Помогает macOS About, системному микшеру звука и т.д. Рекомендуется вызывать при старте.  |
| **SDL_GetError**       | Возвращает строку с описанием последней ошибки SDL. Вызывать после `SDL_Init`/`SDL_CreateWindow` и т.д., если функция вернула ошибку.                 |

## Окно

| Термин                        | Объяснение                                                                                                                                                 |
|-------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **SDL_Window**                | Непрозрачный handle окна. Создаётся через `SDL_CreateWindow`, передаётся в `SDL_Vulkan_CreateSurface` и `SDL_DestroyWindow`.                               |
| **SDL_WindowID**              | Уникальный ID окна (`Uint32`). Ноль — невалидный ID. Используется в событиях (`event.window.windowID`).                                                    |
| **SDL_WindowFlags**           | Флаги окна для `SDL_CreateWindow`. Список — в [Справочнике API](api-reference.md#sdl_windowflags-часто-используемые).                                      |
| **SDL_WINDOW_VULKAN**         | Окно создаётся с поддержкой Vulkan. Обязателен для `SDL_Vulkan_CreateSurface`. При создании такого окна SDL может загрузить Vulkan loader.                 |
| **SDL_GetWindowSize**         | Возвращает размер окна в **оконных координатах** (логические единицы).                                                                                     |
| **SDL_GetWindowSizeInPixels** | Возвращает размер клиентской области в **пикселях**. Для Vulkan swapchain и framebuffer нужен именно он. На HiDPI может отличаться от `SDL_GetWindowSize`. |

## События

| Термин                                  | Объяснение                                                                                                                                                                                                                            |
|-----------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **SDL_Event**                           | Union всех типов событий. `event.type` определяет тип (`SDL_EVENT_QUIT`, `SDL_EVENT_KEY_DOWN` и т.д.).                                                                                                                                |
| **SDL_EventType**                       | Перечисление типов: `SDL_EVENT_QUIT`, `SDL_EVENT_WINDOW_CLOSE_REQUESTED`, `SDL_EVENT_KEY_DOWN`, `SDL_EVENT_WINDOW_*` и др.                                                                                                            |
| **SDL_EVENT_QUIT**                      | Глобальный запрос выхода приложения (обычно при закрытии последнего окна или Alt+F4). Обработать и вернуть `SDL_APP_SUCCESS`.                                                                                                         |
| **SDL_EVENT_WINDOW_CLOSE_REQUESTED**    | Менеджер окон запрашивает закрытие конкретного окна (крестик). Приходит **до** `SDL_EVENT_QUIT`. Для Vulkan: сначала уничтожить swapchain, затем окно.                                                                                |
| **SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED** | Размер окна в пикселях изменился (resize, смена DPI). Использовать для пересоздания Vulkan swapchain. `event.window.data1` и `data2` — ширина и высота в пикселях.                                                                    |
| **SDL_EVENT_KEY_DOWN**                  | Клавиша нажата. `event.key.key` — виртуальный код (`SDL_Keycode`, зависит от раскладки); `event.key.scancode` — физическая клавиша (не зависит от раскладки; для WASD предпочтительнее). `event.key.repeat` — `true` при автоповторе. |
| **SDL_PollEvent**                       | Извлекает одно событие из очереди. В режиме `SDL_MAIN_USE_CALLBACKS` не вызывать — события приходят в `SDL_AppEvent`.                                                                                                                 |

## Main callbacks

| Термин                     | Объяснение                                                                                                                                                                    |
|----------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **SDL_MAIN_USE_CALLBACKS** | Макрос. Определить до `#include "SDL3/SDL_main.h"`. Вместо `main()` приложение реализует `SDL_AppInit`, `SDL_AppEvent`, `SDL_AppIterate`, `SDL_AppQuit`.                      |
| **SDL_MAIN_HANDLED**       | Макрос. Приложение само предоставляет entry point (WinMain и т.д.) и вызывает `SDL_SetMainReady()` или `SDL_RunApp()`.                                                        |
| **SDL_AppInit**            | Callback инициализации. Вызывается один раз при старте. Создать окно, вернуть `SDL_APP_CONTINUE` для продолжения.                                                             |
| **SDL_AppEvent**           | Callback события. Вызывается для каждого события. Обработать (например, `SDL_EVENT_QUIT` → вернуть `SDL_APP_SUCCESS`).                                                        |
| **SDL_AppIterate**         | Callback одного кадра. Вызывается повторно (игровой цикл). Рисовать, обновлять логику.                                                                                        |
| **SDL_AppQuit**            | Callback завершения. Вызывается перед выходом. Освободить ресурсы.                                                                                                            |
| **SDL_AppResult**          | Результат callback: `SDL_APP_CONTINUE` (продолжить), `SDL_APP_SUCCESS` (завершить успешно), `SDL_APP_FAILURE` (завершить с ошибкой).                                          |
| **appstate**               | Указатель, опционально записываемый в `*appstate` в `SDL_AppInit`. Передаётся во все остальные callbacks. Удобен для хранения состояния приложения без глобальных переменных. |

## Vulkan

| Термин                                  | Объяснение                                                                                                                                                                        |
|-----------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **SDL_Vulkan_GetInstanceExtensions**    | Возвращает массив имён расширений Vulkan, необходимых для создания surface на SDL-окне. Передать в `VkInstanceCreateInfo::ppEnabledExtensionNames`. Сигнатура: `(Uint32* count)`. |
| **SDL_Vulkan_CreateSurface**            | Создаёт `VkSurfaceKHR` для окна. Окно должно быть с `SDL_WINDOW_VULKAN`, instance — с расширениями из `SDL_Vulkan_GetInstanceExtensions`.                                         |
| **SDL_Vulkan_DestroySurface**           | Уничтожает surface. Вызывать **до** `SDL_DestroyWindow`.                                                                                                                          |
| **SDL_Vulkan_GetVkGetInstanceProcAddr** | Возвращает указатель на `vkGetInstanceProcAddr`. Можно передать в `volkInitializeCustom()` вместо `volkInitialize()`, если SDL уже загрузил Vulkan loader.                        |
| **SDL_Vulkan_LoadLibrary**              | Явная загрузка Vulkan loader (опционально). SDL загружает его при создании окна с `SDL_WINDOW_VULKAN`.                                                                            |
| **SDL_Vulkan_UnloadLibrary**            | Выгрузка Vulkan loader (парная к `SDL_Vulkan_LoadLibrary`). Вызывать после уничтожения всех Vulkan-ресурсов и окон.                                                               |
| **SDL_Vulkan_GetPresentationSupport**   | Проверяет поддержку presentation для данной queue family. Полезно при выборе `VkPhysicalDevice`.                                                                                  |
| **VkSurfaceKHR**                        | Vulkan-поверхность. Связывает Vulkan с окном. Создаётся через `SDL_Vulkan_CreateSurface` или напрямую `vkCreate*SurfaceKHR`.                                                      |

## Hints

| Термин                      | Объяснение                                                                                                                              |
|-----------------------------|-----------------------------------------------------------------------------------------------------------------------------------------|
| **SDL_HINT_VULKAN_LIBRARY** | Путь к Vulkan loader. Задать через `SDL_SetHint` до создания Vulkan-окна или `SDL_Vulkan_LoadLibrary`.                                  |
| **SDL_HINT_VULKAN_DISPLAY** | Индекс дисплея для `SDL_Vulkan_CreateSurface` — **строка** (`"0"`, `"1"` и т.д.). По умолчанию 0. Задать до `SDL_Vulkan_CreateSurface`. |
