## Базовые понятия аудио

| Термин                | Пояснение                                                                                                                                                                                                                                  |
|-----------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Sample**            | Один отсчёт аудиоданных. Для формата f32 — одно число 32-bit float. Для s16 — одно число 16-bit signed integer.                                                                                                                            |
| **Frame / PCM Frame** | Группа сэмплов, равная числу каналов. Стерео: 1 frame = 2 sample (левый + правый). Моно: 1 frame = 1 sample. 5.1: 1 frame = 6 samples. «Frame» и «PCM frame» в miniaudio — одно и то же. Не путать с сжатым кадром (например, FLAC frame). |
| **Channel**           | Поток монофонического аудио с одного динамика или в один микрофон. Стерео = 2 канала, 5.1 = 6 каналов.                                                                                                                                     |
| **Sample Rate**       | Частота дискретизации в Hz. Рекомендованный диапазон 8000–384000 Hz. 0 = native устройства. Типичные значения: 44100, 48000. Число PCM frames в секунду.                                                                                   |
| **Format**            | Формат сэмпла: `ma_format_f32`, `ma_format_s16`, `ma_format_s24`, `ma_format_s32`, `ma_format_u8`. Все native-endian.                                                                                                                      |
| **Interleaved**       | Сэмплы каналов в одном frame идут подряд: для стерео L,R,L,R, … В miniaudio все данные interleaved.                                                                                                                                        |

## Структуры и объекты

| Термин                    | Пояснение                                                                                                                                                                                                                            |
|---------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Transparent structure** | Объекты miniaudio — прозрачные структуры C. Нет opaque handle; вы выделяете память сами (`ma_device device;` или `malloc`). Адрес объекта не должен меняться в течение жизненного цикла.                                             |
| **Config/init pattern**   | Паттерн инициализации: сначала заполняете config (`ma_device_config_init`, `ma_engine_config_init`), затем вызываете init (`ma_device_init`, `ma_engine_init`) с этим config. Config можно держать на стеке и не хранить после init. |
| **ma_result**             | Код результата. `MA_SUCCESS` — успех. Другие: `MA_INVALID_ARGS`, `MA_NO_MEMORY`, `MA_DEVICE_IN_USE` и т.д.                                                                                                                           |
| **Callback**              | Функция, которую вы реализуете, а miniaudio вызывает. Например, `data_callback` — вызывается когда нужно заполнить/прочитать буфер устройства.                                                                                       |

## Low-level API

| Термин                           | Пояснение                                                                                                                                                                                                             |
|----------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **ma_context**                   | Объект контекста бэкенда. Представляет платформенный аудио-бэкенд. Используется для перечисления устройств (`ma_context_get_devices`) и передачи в `ma_device_init`. NULL — miniaudio создаёт контекст автоматически. |
| **ma_context_init**              | Инициализирует контекст. Первый параметр — массив бэкендов или NULL (приоритеты по умолчанию).                                                                                                                        |
| **ma_context_get_devices**       | Возвращает списки устройств playback и capture. Указатели на внутренние буферы miniaudio — не освобождать.                                                                                                            |
| **ma_context_enumerate_devices** | Альтернатива `ma_context_get_devices` — перечисление через callback, без внутренней heap-аллокации.                                                                                                                   |
| **ma_device**                    | Устройство воспроизведения или захвата. Связь с физическим устройством (динамики, микрофон).                                                                                                                          |
| **ma_device_config**             | Конфигурация устройства. Инициализируется через `ma_device_config_init(ma_device_type_playback)` или `ma_device_type_capture`, `ma_device_type_duplex`.                                                               |
| **ma_device_init**               | Инициализирует устройство. Параметры: context (или NULL), config, указатель на ma_device.                                                                                                                             |
| **ma_device_start**              | Запускает устройство. По умолчанию устройство в состоянии stopped.                                                                                                                                                    |
| **ma_device_stop**               | Останавливает устройство.                                                                                                                                                                                             |
| **ma_device_uninit**             | Освобождает устройство. Останавливает его, если было запущено.                                                                                                                                                        |
| **Period / Buffer**              | Размер порции данных в PCM frames или миллисекундах. `periodSizeInFrames`, `periodSizeInMilliseconds`, `periods` в config. Меньший period — меньше latency, выше нагрузка на CPU.                                     |
| **ma_performance_profile**       | Подсказка по производительности: `ma_performance_profile_low_latency` (default) или `ma_performance_profile_conservative`. Влияет на размер буфера по умолчанию.                                                      |
| **ma_share_mode**                | Режим доступа к устройству: shared (несколько приложений) или exclusive (эксклюзивный). Зависит от бэкенда.                                                                                                           |
| **notificationCallback**         | Callback при изменении состояния устройства (start, stop, reroute). Опционально в `ma_device_config`.                                                                                                                 |
| **data_callback**                | Callback, вызываемый когда miniaudio нужны данные (playback: записать в `pOutput`) или когда есть новые данные (capture: прочитать из `pInput`). Playback: `pInput` = NULL. Capture: `pOutput` = NULL.                |
| **ma_device_info**               | Информация об устройстве. Содержит `id` (для config), `name` (для UI).                                                                                                                                                |
| **ma_device_type**               | Тип устройства: `ma_device_type_playback`, `ma_device_type_capture`, `ma_device_type_duplex`, `ma_device_type_loopback` (только WASAPI).                                                                              |
| **ma_backend**                   | Бэкенд платформы: WASAPI, DirectSound, WinMM, Core Audio, ALSA, PulseAudio, JACK и т.д.                                                                                                                               |

## High-level API

| Термин                      | Пояснение                                                                                                                                  |
|-----------------------------|--------------------------------------------------------------------------------------------------------------------------------------------|
| **ma_engine**               | Высокоуровневый движок. Включает device, resource manager и node graph. Простой способ воспроизводить звуки.                               |
| **ma_engine_init**          | Инициализирует engine. NULL config — значения по умолчанию.                                                                                |
| **ma_engine_play_sound**    | Воспроизводит «inline» звук один раз. Файл загружается и воспроизводится, затем звук переиспользуется.                                     |
| **ma_engine_uninit**        | Освобождает engine.                                                                                                                        |
| **ma_sound**                | Объект звука. Представляет один экземпляр воспроизведения. Для нескольких одновременных воспроизведений одного файла — несколько ma_sound. |
| **ma_sound_init_from_file** | Инициализирует звук из файла.                                                                                                              |
| **ma_sound_start**          | Запускает воспроизведение. Звук по умолчанию не запущен.                                                                                   |
| **ma_sound_stop**           | Останавливает воспроизведение. Не перематывает к началу.                                                                                   |
| **ma_sound_uninit**         | Освобождает звук.                                                                                                                          |
| **ma_sound_group**          | Группа звуков. Отдельная громкость и эффекты (например, SFX, музыка, голос).                                                               |
| **ma_resource_manager**     | Менеджер ресурсов. Загрузка и стриминг звуков, подсчёт ссылок.                                                                             |
| **ma_node_graph**           | Граф узлов для микширования и эффектов. Engine — это node graph.                                                                           |

## Data sources и декодеры

| Термин                             | Пояснение                                                                                           |
|------------------------------------|-----------------------------------------------------------------------------------------------------|
| **ma_data_source**                 | Абстракция источника аудиоданных. `ma_decoder`, `ma_noise`, `ma_waveform` реализуют этот интерфейс. |
| **ma_data_source_read_pcm_frames** | Читает PCM frames из data source.                                                                   |
| **ma_decoder**                     | Декодер аудиофайлов. WAV, FLAC, MP3 — встроены.                                                     |
| **ma_decoder_init_file**           | Инициализирует декодер из файла.                                                                    |
| **ma_decoder_read_pcm_frames**     | Читает декодированные PCM frames.                                                                   |
| **ma_decoder_uninit**              | Освобождает декодер.                                                                                |
| **pUserData**                      | Указатель, передаваемый в config. Доступен в callback через `pDevice->pUserData`.                   |

## Результаты и форматы

| Термин                | Пояснение                                                                                                                                                                                              |
|-----------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **MA_SUCCESS**        | Успешное выполнение.                                                                                                                                                                                   |
| **ma_format**         | Формат сэмпла: `ma_format_f32` (float [-1,1]), `ma_format_s16` (int16), `ma_format_s24`, `ma_format_s32`, `ma_format_u8` (uint8). Подробнее: [Справочник API — ma_format](api-reference.md#ma_format). |
| **ma_format_unknown** | Использовать формат устройства по умолчанию (в config).                                                                                                                                                |
| **MA_MAX_CHANNELS**   | Максимальное число каналов. Значение: **254** (см. [miniaudio.h](../../external/miniaudio/miniaudio.h)).                                                                                               |
