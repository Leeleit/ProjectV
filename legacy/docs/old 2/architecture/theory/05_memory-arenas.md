# Управление Памятью: Арены и Аллокаторы

В C++ `new` и `delete` — это медленные операции. В игровом цикле (Game Loop) они недопустимы.
Почему?

1. **Поиск свободного блока:** `malloc` ищет место в куче (Heap), это долго (O(n)).
2. **Фрагментация:** Память становится похожа на швейцарский сыр, где нет места для больших объектов.
3. **Кэш-промахи:** Разбросанные объекты убивают производительность.

В ProjectV мы используем **Арены (Arenas)**. Это позволяет нам управлять памятью так же эффективно, как стек.

## 1. Линейный Аллокатор (Linear / Arena Allocator)

Представьте стопку тарелок.

* **Allocate:** Взять верхнюю тарелку (сдвинуть указатель `ptr += size`). Это мгновенно (O(1)).
* **Reset:** Убрать все тарелки разом (вернуть указатель в начало `ptr = start`). Это мгновенно.

Мы выделяем большой блок памяти (напр. 100 МБ) на старте и "нарезаем" его сами.

### Принцип работы

1. Выделяем блок памяти один раз (Start, End).
2. При запросе памяти проверяем, есть ли место.
3. Возвращаем текущий указатель `ptr`.
4. Сдвигаем `ptr` на размер объекта.
5. В конце кадра (или уровня) сбрасываем `ptr` в начало.

### Где используем:

* **Frame Arena:** Временные данные, живущие 1 кадр. (Сбрасывается в конце кадра).
* **Chunk Arena:** Данные вокселей. (Сбрасывается при выгрузке уровня).

## 2. Стек-Аллокатор (Stack Allocator)

Как Линейный, но можно освобождать память LIFO (Last In, First Out).

* **Push:** Запомнить текущую позицию (`Marker`).
* **Pop:** Вернуть указатель на `Marker`.

### Где используем:

* Временные буферы внутри функций.
* Рекурсивные алгоритмы (напр. обход октодерева).

## 3. Пул Объектов (Object Pool)

Для объектов фиксированного размера (частицы, пули).

* Память выделена заранее массивом.
* Список свободных слотов (Free List).
* **Acquire:** Взять слот из Free List (O(1)).
* **Release:** Вернуть слот в Free List (O(1)).

### Где используем:

* Частицы (Particles).
* Снаряды (Projectiles).
* Компоненты ECS (внутри Flecs).

## 4. Стратегия ProjectV для Вокселей

### Пример: Загрузка Чанка

**Плохо (Традиционный подход):**

```cpp
VoxelChunk* chunk = new VoxelChunk(); // Аллокация 1
chunk->voxels = new Voxel[...];       // Аллокация 2
chunk->mesh = new Mesh();             // Аллокация 3
// Результат: 3 аллокации, фрагментация, медленно.
```

**Хорошо (ProjectV подход):**

```cpp
// Выделяем память из ChunkArena одним блоком
void* block = chunkArena.allocate(TotalSize);
// Размещаем объекты внутри блока (Placement New)
VoxelChunk* chunk = new (block) VoxelChunk();
// Результат: 1 аллокация (сдвиг указателя), данные лежат подряд.
```

### Правила:

1. **Минимизируй `new`**: В главном цикле `new` запрещен.
2. **Используй Арены**: Для всего, что имеет четкое время жизни (Level, Frame).
3. **Используй Пулы**: Для часто создаваемых/удаляемых объектов.
4. **Стандартные контейнеры**: `std::vector` с кастомным аллокатором (Polymorphic Allocator / PMR).
