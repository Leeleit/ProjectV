## Общие термины Vulkan

| Термин                        | Объяснение                                                                                                                                                                                                          |
|-------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Loader**                    | Системная библиотека (`vulkan-1.dll` на Windows, `libvulkan.so` на Linux). Связывает приложение с драйвером GPU. Загружает функции Vulkan по имени во время выполнения.                                             |
| **Entrypoint**                | Указатель на функцию Vulkan (например, `vkCreateInstance`, `vkCmdDraw`). В Vulkan функции не линкуются статически — их загружают через loader по имени строки. Entrypoint — это полученный указатель.               |
| **Dispatch**                  | Цепочка вызовов: ваша программа → loader → validation layer (если есть) → драйвер GPU. Каждый переход добавляет накладные расходы. volk позволяет получать указатели напрямую на драйвер, минуя лишние уровни.      |
| **PFN_vkGetInstanceProcAddr** | Тип указателя на функцию `vkGetInstanceProcAddr`. Эту функцию предоставляет loader; она позволяет по имени строки получить указатель на любую instance-функцию Vulkan. volk использует её для загрузки entrypoints. |
| **VK_NO_PROTOTYPES**          | Макрос Vulkan, который запрещает объявление прототипов функций в vulkan.h. Требуется для корректной работы volk, чтобы избежать конфликтов линковки с традиционным Vulkan loader.                                   |
| **Validation Layers**         | Слои отладки Vulkan, которые проверяют корректность вызовов API, отслеживают утечки ресурсов и выводят предупреждения. volk полностью совместим с validation layers.                                                |

## Термины volk и мета-загрузчика

| Термин                           | Объяснение                                                                                                                                                                              |
|----------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Мета-загрузчик (Meta-loader)** | Архитектурный паттерн, реализованный volk: библиотека, которая загружает другой загрузчик (Vulkan loader) и управляет указателями на функции, обеспечивая прямой доступ к драйверу GPU. |
| **Direct Driver Calls**          | Прямые вызовы функций драйвера GPU, минуя dispatch цепочку Vulkan loader. volk обеспечивает это через `volkLoadDevice()`, что даёт прирост производительности до 7%.                    |
| **Глобальные указатели функций** | Набор extern-указателей на все функции Vulkan, объявленных в volk.h (например, `PFN_vkCreateInstance vkCreateInstance`). После инициализации volk заполняет эти указатели.              |
| **Header-only режим**            | Режим использования volk, при котором реализация библиотеки содержится в заголовочном файле (`volk.h`). Активируется макросом `VOLK_IMPLEMENTATION`. Удобен для простых проектов.       |
| **volkInitializeCustom**         | Функция volk для инициализации с предоставленным пользователем обработчиком `PFN_vkGetInstanceProcAddr`. Используется при интеграции с SDL3, где loader уже загружен SDL.               |

## Функции инициализации volk

| Термин                               | Объяснение                                                                                                                                                                      |
|--------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **volkInitialize()**                 | Базовая функция инициализации volk. Загружает Vulkan loader, получает `vkGetInstanceProcAddr` и загружает глобальные instance-функции. Должна вызываться до `vkCreateInstance`. |
| **volkLoadInstance(VkInstance)**     | Загружает все instance-уровневые функции Vulkan после создания `VkInstance`. Заполняет глобальные указатели на функции, которые требуют instance (например, `vkCreateDevice`).  |
| **volkLoadDevice(VkDevice)**         | Загружает device-уровневые функции Vulkan после создания `VkDevice`. Указатели направляются напрямую в драйвер GPU, минуя loader dispatch. Критично для производительности.     |
| **volkLoadInstanceOnly(VkInstance)** | Альтернатива `volkLoadInstance()`, которая загружает только instance-функции, но не device-функции. Позволяет отложить загрузку device-функций до создания устройства.          |
| **volkFinalize()**                   | Выгружает Vulkan loader и сбрасывает все глобальные указатели в NULL. Редко требуется, но может быть полезна для повторной инициализации или освобождения ресурсов.             |
| **volkGetInstanceVersion()**         | Возвращает версию Vulkan, поддерживаемую loader (например, `VK_API_VERSION_1_3`). Возвращает 0, если `volkInitialize()` не был вызван или завершился ошибкой.                   |

## Таблицы функций и указатели

| Термин                      | Объяснение                                                                                                                                                    |
|-----------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **VolkInstanceTable**       | Структура, содержащая указатели на все instance-функции Vulkan. Позволяет работать с несколькими `VkInstance` одновременно, изолируя их функции.              |
| **VolkDeviceTable**         | Структура, содержащая указатели на все device-функции Vulkan. Необходима для приложений с несколькими `VkDevice` (например, async compute + graphics queues). |
| **volkLoadInstanceTable()** | Загружает указатели функций в `VolkInstanceTable` для заданного `VkInstance`. Альтернатива использованию глобальных указателей, обеспечивает лучшую изоляцию. |
| **volkLoadDeviceTable()**   | Загружает указатели функций в `VolkDeviceTable` для заданного `VkDevice`. Рекомендуется для многопоточных приложений и при работе с несколькими GPU.          |
| **volkGetLoadedInstance()** | Возвращает последний `VkInstance`, переданный в `volkLoadInstance()`. Полезно для отладки и проверки состояния инициализации.                                 |
| **volkGetLoadedDevice()**   | Возвращает последний `VkDevice`, переданный в `volkLoadDevice()`. Используется для проверки корректности загрузки device-функций.                             |

## Термины производительности и интеграции

| Термин                               | Объяснение                                                                                                                                                                  |
|--------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Dispatch Overhead**                | Накладные расходы времени CPU, возникающие при прохождении вызова через цепочку loader → validation layers → драйвер. volk устраняет до 7% этого overhead.                  |
| **Async Compute Queue**              | Отдельная очередь GPU для выполнения compute-шейдеров параллельно с графическим рендерингом. volk оптимизирован для работы с несколькими очередями через `VolkDeviceTable`. |
| **vmaImportVulkanFunctionsFromVolk** | Функция VMA (Vulkan Memory Allocator), которая импортирует указатели на Vulkan функции из volk. Обеспечивает интеграцию volk с управлением памятью GPU.                     |
| **Tracy Profiling Integration**      | Возможность передачи указателей volk в Tracy для точного измерения времени выполнения Vulkan вызовов. Использует `volkGetInstanceProcAddr` для получения указателей.        |
| **Graceful Degradation**             | Стратегия volk при отсутствии Vulkan loader: возврат ошибки `VK_ERROR_INITIALIZATION_FAILED` вместо краха приложения. Позволяет реализовать fallback на другие API.         |
| **Platform-Specific Defines**        | Макросы платформы, которые volk определяет автоматически (`VK_USE_PLATFORM_WIN32_KHR`, `VK_USE_PLATFORM_XLIB_KHR` и др.). Упрощают кроссплатформенную сборку.               |
| **CMake targets**                    | Предоставляемые volk CMake цели: `volk` (статическая библиотека) и `volk_headers` (интерфейсный target для header-only режима).                                             |
| **VOLK_NAMESPACE**                   | Опция для помещения символов volk в namespace `volk::`, предотвращающая конфликты символов при смешивании volk с прямым использованием Vulkan.                              |
| **VOLK_NO_DEVICE_PROTOTYPES**        | Макрос для скрытия прототипов device-функций, позволяющий компилятору обнаруживать ошибки использования device-функций без предварительной загрузки через `volkLoadDevice`. |
