# Глоссарий GLM

🟢 **Уровень 1: Начинающий**

Словарь терминов GLM и математики для графики.

## Типы данных

| Термин                  | Объяснение                                                                                                                          |
|-------------------------|-------------------------------------------------------------------------------------------------------------------------------------|
| **glm**                 | OpenGL Mathematics — header-only библиотека, повторяющая типы и функции GLSL в C++. Векторы, матрицы, кватернионы, трансформации.   |
| **GLSL**                | OpenGL Shading Language. Язык шейдеров; GLM следует его именованию и соглашениям (column-major матрицы).                            |
| **vec2, vec3, vec4**    | Векторы из 2, 3 и 4 компонент (float). vec3 — позиция, направление; vec2 — UV, размер; vec4 — цвет с альфой, однородная координата. |
| **mat3, mat4**          | Квадратные матрицы 3×3 и 4×4. mat4 — преобразование позиции вершины: модель, вид, проекция.                                         |
| **quat**                | Кватернион — представление поворота без gimbal lock. Тип `glm::quat`, функции: `slerp`, `mat4_cast` — в gtc/quaternion.hpp.         |
| **ivec2, ivec3, ivec4** | Целочисленные векторы. Индексы, целочисленные координаты.                                                                           |
| **uvec2, uvec3, uvec4** | Беззнаковые целочисленные векторы. Хеши, флаги.                                                                                     |
| **dvec2, dvec3, dvec4** | Векторы двойной точности (double).                                                                                                  |
| **bvec2, bvec3, bvec4** | Булевы векторы. Результаты сравнений.                                                                                               |

## Матричные операции

| Термин               | Объяснение                                                                                        |
|----------------------|---------------------------------------------------------------------------------------------------|
| **column-major**     | Порядок хранения матрицы по столбцам. В GLM и GLSL матрицы column-major; первый индекс — столбец. |
| **glm::translate**   | Строит матрицу переноса. Использование: `glm::translate(mat4(1.0f), vec3(x, y, z))`.              |
| **glm::rotate**      | Строит матрицу поворота вокруг оси на угол (в радианах).                                          |
| **glm::scale**       | Строит матрицу масштабирования по осям.                                                           |
| **glm::perspective** | Строит матрицу перспективной проекции (FOV в радианах, aspect, near, far).                        |
| **glm::ortho**       | Строит матрицу ортогональной проекции (лево, право, низ, верх, near, far).                        |
| **glm::lookAt**      | Строит матрицу вида камеры: позиция камеры, точка взгляда, вектор «вверх».                        |
| **glm::inverse**     | Обратная матрица.                                                                                 |
| **glm::transpose**   | Транспонирование матрицы.                                                                         |
| **glm::determinant** | Определитель матрицы.                                                                             |

## Векторные операции

| Термин             | Объяснение                                   |
|--------------------|----------------------------------------------|
| **glm::dot**       | Скалярное произведение векторов.             |
| **glm::cross**     | Векторное произведение (только для vec3).    |
| **glm::normalize** | Нормализация вектора (приведение к длине 1). |
| **glm::length**    | Длина вектора.                               |
| **glm::distance**  | Расстояние между двумя точками.              |
| **glm::reflect**   | Отражение вектора относительно нормали.      |
| **glm::refract**   | Преломление вектора (закон Снеллиуса).       |

## Кватернионы

| Термин             | Объяснение                                                                                                    |
|--------------------|---------------------------------------------------------------------------------------------------------------|
| **glm::angleAxis** | Создание кватерниона из угла и оси.                                                                           |
| **glm::slerp**     | Сферическая линейная интерполяция кватернионов (плавный поворот).                                             |
| **glm::mix**       | Линейная интерполяция кватернионов (быстрее, менее точно для больших углов).                                  |
| **glm::mat4_cast** | Преобразование кватерниона в матрицу 4×4.                                                                     |
| **glm::quat_cast** | Преобразование матрицы в кватернион.                                                                          |
| **gimbal lock**    | «Складывание рамок» — потеря степени свободы при использовании углов Эйлера. Кватернионы решают эту проблему. |

## Передача данных

| Термин                        | Объяснение                                                                                                                 |
|-------------------------------|----------------------------------------------------------------------------------------------------------------------------|
| **glm::value_ptr**            | Возвращает указатель на непрерывную память вектора или матрицы. Нужен для записи в uniform-буфер GPU. Из gtc/type_ptr.hpp. |
| **glm::make_vec3, make_mat4** | Создают GLM-тип из указателя на непрерывную память. Для чтения из GPU.                                                     |
| **uniform-буфер**             | Данные, передаваемые в шейдер (например, матрица модели). В C++ записываются через value_ptr.                              |
| **std140**                    | Стандарт выравнивания данных для uniform-буферов. vec3 занимает 16 байт, mat4 — 64 байта с выравниванием 16.               |

## Системы координат

| Термин             | Объяснение                                                                                                                 |
|--------------------|----------------------------------------------------------------------------------------------------------------------------|
| **NDC**            | Normalized Device Coordinates — координаты вершины после проекции. X, Y: [-1, 1]; Z: [-1, 1] (OpenGL) или [0, 1] (Vulkan). |
| **Правая система** | Система координат OpenGL: большой палец = X, указательный = Y, средний = Z.                                                |
| **Левая система**  | Система координат DirectX. Переключение: `GLM_FORCE_LEFT_HANDED`.                                                          |
| **radians**        | Углы в радианах. GLM использует радианы для всех углов.                                                                    |

## Расширения GLM

| Термин  | Объяснение                                                                                   |
|---------|----------------------------------------------------------------------------------------------|
| **EXT** | Стабильные расширения glm (ext/*). Например, ext/matrix_transform.hpp.                       |
| **GTC** | Рекомендуемые расширения (gtc/*): стабильные, с тестами. Например, gtc/quaternion.hpp.       |
| **GTX** | Экспериментальные расширения (gtx/*). Требуют `GLM_ENABLE_EXPERIMENTAL`. API может меняться. |

## Макросы конфигурации

| Термин                          | Объяснение                                                     |
|---------------------------------|----------------------------------------------------------------|
| **GLM_FORCE_RADIANS**           | Использовать радианы для всех углов.                           |
| **GLM_FORCE_DEPTH_ZERO_TO_ONE** | Диапазон глубины [0, 1] (Vulkan) вместо [-1, 1] (OpenGL).      |
| **GLM_FORCE_LEFT_HANDED**       | Левая система координат (DirectX).                             |
| **GLM_FORCE_INTRINSICS**        | Включить SIMD оптимизации.                                     |
| **GLM_FORCE_SSE2 / AVX / AVX2** | Принудительно использовать конкретные SIMD инструкции.         |
| **GLM_FORCE_ALIGNED_GENTYPES**  | Использовать выровненные типы (для SIMD).                      |
| **GLM_FORCE_EXPLICIT_CTOR**     | Запретить неявные преобразования типов.                        |
| **GLM_FORCE_SIZE_T_LENGTH**     | `.length()` возвращает `size_t` вместо `int`.                  |
| **GLM_ENABLE_EXPERIMENTAL**     | Разрешить включение экспериментальных расширений GTX.          |
| **GLM_FORCE_SWIZZLE**           | Включить swizzle-операторы (.xyz, .rgb). Замедляет компиляцию. |

## Графические концепции

| Термин                | Объяснение                                                             |
|-----------------------|------------------------------------------------------------------------|
| **Model matrix**      | Матрица модели: локальные координаты объекта → мировые координаты.     |
| **View matrix**       | Матрица вида: мировые координаты → координаты относительно камеры.     |
| **Projection matrix** | Матрица проекции: 3D пространство → 2D экран.                          |
| **MVP**               | Model-View-Projection: `proj * view * model`. Порядок умножения важен! |
| **FOV**               | Field of View — поле зрения камеры (в радианах для GLM).               |
| **Aspect ratio**      | Соотношение сторон экрана (ширина / высота).                           |
| **Near / Far plane**  | Ближняя и дальняя плоскости отсечения.                                 |

## Дополнительные функции

| Термин              | Объяснение                                                                                                       |
|---------------------|------------------------------------------------------------------------------------------------------------------|
| **glm::mix**        | Линейная интерполяция: `mix(a, b, t) = a + t*(b-a)`. Работает со скалярами, векторами, матрицами, кватернионами. |
| **glm::clamp**      | Ограничение значения между min и max: `clamp(x, min, max)`.                                                      |
| **glm::step**       | Ступенчатая функция: 0 если x < edge, иначе 1.                                                                   |
| **glm::smoothstep** | Сглаженная ступенчатая функция (Hermite).                                                                        |
| **glm::radians**    | Перевод градусов в радианы.                                                                                      |
| **glm::degrees**    | Перевод радианов в градусы.                                                                                      |
