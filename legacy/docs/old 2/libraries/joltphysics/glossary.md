## Базовые понятия физики

| Термин                                       | Объяснение                                                                                                                                        |
|----------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------|
| **Физическая симуляция**                     | Процесс вычисления движения и взаимодействия объектов по законам физики. В реальном времени обновляется каждый кадр.                              |
| **Детерминированность**                      | Свойство симуляции давать одинаковые результаты при одинаковых начальных условиях и входных данных. Критично для сетевых игр и реплеев.           |
| **Дискретная симуляция**                     | Симуляция, обновляемая дискретными шагами (кадры) с фиксированным или переменным временным шагом.                                                 |
| **Непрерывная симуляция**                    | Теоретическая модель, где время течёт непрерывно. В играх аппроксимируется дискретными шагами.                                                    |
| **Широкоя фаза (Broad Phase)**               | Этап обнаружения коллизий, определяющий, какие пары тел потенциально могут столкнуться. Использует пространственное разбиение (AABB, grid, tree). |
| **Узкая фаза (Narrow Phase)**                | Этап точного определения коллизий между парами тел, найденных широкой фазой. Вычисляет точки контакта, нормали, проникновение.                    |
| **Решатель ограничений (Constraint Solver)** | Алгоритм, вычисляющий силы и импульсы для удовлетворения ограничений (контакты, шарниры и т.д.).                                                  |
| **Интегратор**                               | Алгоритм, обновляющий позиции и скорости тел на основе сил и ускорений (обычно Verlet или Euler).                                                 |

## Архитектура JoltPhysics

| Термин                         | Объяснение                                                                                                                                                                               |
|--------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **PhysicsSystem**              | Центральный класс JoltPhysics, управляющий всей физической симуляцией. Содержит тела, формы, коллизии, ограничения. Инициализируется через `PhysicsSystem::Init()`.                      |
| **BodyInterface**              | Интерфейс для работы с физическими телами: создание (`CreateBody()`), удаление (`DestroyBody()`), установка свойств, применение сил. Получить через `PhysicsSystem::GetBodyInterface()`. |
| **Shape**                      | Геометрическая форма тела (куб, сфера, капсула и т.д.). Формы неизменяемы (immutable) и могут использоваться несколькими телами. Все формы наследуются от `JPH::Shape`.                  |
| **BodyID**                     | 32-битный идентификатор тела (16 бит — индекс, 16 бит — sequence number). Используется вместо указателей для безопасности. `BodyID::cInvalidBodyID` означает невалидный ID.              |
| **BodyCreationSettings**       | Структура с настройками для создания тела: позиция, вращение, тип движения, форма, слои, физические свойства.                                                                            |
| **TempAllocator**              | Аллокатор временной памяти для физических расчётов. Выделяет память на стеке или в пуле для минимизации аллокаций кучи. Используется в `PhysicsSystem::Update()`.                        |
| **JobSystem**                  | Система задач для многопоточного выполнения физических вычислений. JoltPhysics использует пул потоков с work-stealing. `JobSystemThreadPool` — стандартная реализация.                   |
| **Factory**                    | Фабрика для создания стандартных форм и ограничений. Должна быть создана до регистрации типов. Доступ через `JPH::Factory::sInstance`.                                                   |
| **RegisterDefaultAllocator()** | Первая функция, которую нужно вызвать перед использованием JoltPhysics. Регистрирует аллокатор по умолчанию.                                                                             |
| **RegisterTypes()**            | Регистрирует все стандартные типы форм и ограничений. Вызывать после создания Factory.                                                                                                   |
| **ShapeResult**                | Результат создания формы через `ShapeSettings::Create()`. Содержит указатель на форму и информацию об ошибках. Проверять через `IsValid()`.                                              |
| **MassProperties**             | Массовые свойства тела: масса, тензор инерции, центр масс. Вычисляются автоматически из формы или задаются вручную.                                                                      |

## Тела и формы

| Термин                                   | Объяснение                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Физическое тело (Body)**               | Объект в физической симуляции, имеющий массу, инерцию, позицию, вращение, скорость. Управляется через `BodyInterface`.                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **Тип движения (Motion Type)**           | Определяет, как тело движется: `Static` (неподвижно), `Kinematic` (движется по заданному пути), `Dynamic` (реагирует на силы). Задаётся в `BodyCreationSettings`.                                                                                                                                                                                                                                                                                                                                                                            |
| **Масса (Mass)**                         | Мера инерции тела. Влияет на реакцию на силы. Вычисляется автоматически из формы и плотности, либо задаётся вручную через `MassProperties`.                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Инерция (Inertia)**                    | Сопротивление тела изменению вращательного движения. Тензор инерции зависит от формы и распределения массы. Для сложных форм вычисляется автоматически.                                                                                                                                                                                                                                                                                                                                                                                      |
| **Центр масс (Center of Mass)**          | Точка, через которую проходит равнодействующая сил инерции. В JoltPhysics обычно совпадает с центром формы. Получить через `BodyInterface::GetCenterOfMassPosition()`.                                                                                                                                                                                                                                                                                                                                                                       |
| **Форма (Shape)**                        | Геометрическое представление тела для коллизий. Все формы наследуются от `JPH::Shape`, неизменяемы (immutable) и могут использоваться несколькими телами. См. [Формы (Shapes)](api-reference.md#формы-shapes).                                                                                                                                                                                                                                                                                                                               |
| **ShapeSettings**                        | Настройки для создания формы. Базовый класс `JPH::ShapeSettings` с методом `Create()`. Для каждого типа формы есть свой класс настроек (`BoxShapeSettings`, `SphereShapeSettings` и т.д.).                                                                                                                                                                                                                                                                                                                                                   |
| **ConvexShape**                          | Выпуклая форма (сфера, куб, капсула, цилиндр, выпуклая оболочка). Имеет лучшую производительность для динамических тел.                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **ConcaveShape**                         | Невыпуклая форма (MeshShape). Используется для статических коллайдеров сложной геометрии.                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Ограничивающий параллелепипед (AABB)** | Axis-Aligned Bounding Box — выровненный по осям ограничивающий прямоугольник, используемый для широкой фазы. Получить через `Shape::GetWorldSpaceBounds()`.                                                                                                                                                                                                                                                                                                                                                                                  |
| **Составная форма (Compound Shape)**     | Форма, состоящая из нескольких подформ с собственными трансформациями. Создаётся через `CompoundShapeSettings::AddShape()`.                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **Высотное поле (HeightField)**          | Специальная форма JoltPhysics для представления террейна и воксельного ландшафта. Квадратная сетка высотных сэмплов `sample_count × sample_count`. Использует `HeightFieldShapeSettings` для настройки: `mOffset`, `mScale`, `mSampleCount`, `mBlockSize` (2-8), `mBitsPerSample` (1-8), `mActiveEdgeCosThresholdAngle`. Поддерживает дыры через `HeightFieldShapeConstants::cNoCollisionValue` (FLT_MAX). Форма всегда статична (`MustBeStatic() == true`). Эффективна для больших ландшафтов с разреженным хранением и компрессией данных. |
| **PhysicsMaterial**                      | Физический материал, определяющий свойства поверхности: трение, упругость. Может быть общим для нескольких форм и тел.                                                                                                                                                                                                                                                                                                                                                                                                                       |
| **ConvexRadius**                         | Радиус выпуклости для скругления рёбер форм (`BoxShape`, `CylinderShape`). Улучшает стабильность симуляции, предотвращая "залипание" на острых углах.                                                                                                                                                                                                                                                                                                                                                                                        |

## Коллизии и контакты

| Термин                                          | Объяснение                                                                                                           |
|-------------------------------------------------|----------------------------------------------------------------------------------------------------------------------|
| **Коллизия (Collision)**                        | Событие пересечения двух физических тел.                                                                             |
| **Контакт (Contact)**                           | Точка или область соприкосновения двух тел при коллизии.                                                             |
| **Нормаль контакта (Contact Normal)**           | Вектор, направленный из первого тела во второе, перпендикулярный поверхности контакта.                               |
| **Глубина проникновения (Penetration Depth)**   | Величина, на которую тела проникли друг в друга. Используется для вычисления сил отталкивания.                       |
| **Трение (Friction)**                           | Сила, противодействующая относительному движению поверхностей. Коэффициент трения от 0.0 (скользко) до 1.0 (цепко).  |
| **Упругость (Restitution)**                     | Мера "отскока" при столкновении. Коэффициент упругости от 0.0 (неупругий удар) до 1.0 (идеально упругий).            |
| **Спекулятивный контакт (Speculative Contact)** | Контакт, предсказанный для будущего кадра для предотвраствия туннелирования быстрых объектов.                        |
| **Туннелирование (Tunneling)**                  | Проблема, когда быстрое тело проходит сквозь другое тело за один кадр из-за дискретной симуляции.                    |
| **Непрерывное обнаружение коллизий (CCD)**      | Continuous Collision Detection — техника для предотвращения туннелирования путём трассировки движения между кадрами. |

## Слои и фильтры

| Термин                                    | Объяснение                                                                                                |
|-------------------------------------------|-----------------------------------------------------------------------------------------------------------|
| **Слой объекта (Object Layer)**           | 8-битный идентификатор, определяющий категорию тела (статический, динамический, персонаж, датчик и т.д.). |
| **Слой широкой фазы (Broad Phase Layer)** | 8-битный идентификатор для оптимизации широкой фазы. Группирует объекты с похожими свойствами коллизий.   |
| **Фильтр коллизий (Collision Filter)**    | Правила, определяющие, могут ли два объекта сталкиваться. Основаны на слоях объектов.                     |
| **ObjectVsBroadPhaseLayerFilter**         | Фильтр, определяющий, может ли объект сталкиваться с целым слоем широкой фазы.                            |
| **ObjectVsObjectLayerFilter**             | Фильтр, определяющий, могут ли два конкретных объекта сталкиваться.                                       |
| **Маска коллизий (Collision Mask)**       | Битовые маски для определения, какие слои могут сталкиваться. Альтернатива фильтрам.                      |
| **Датчик (Sensor)**                       | Тело, которое обнаруживает коллизии, но не создаёт сил отталкивания (триггеры).                           |

## Ограничения и моторы

| Термин                                     | Объяснение                                                                                       |
|--------------------------------------------|--------------------------------------------------------------------------------------------------|
| **Ограничение (Constraint)**               | Связь между двумя телами, ограничивающая их относительное движение (шарнир, ползунок и т.д.).    |
| **Шарнир (Hinge)**                         | Ограничение, позволяющее вращение вокруг одной оси (дверь, колено).                              |
| **Ползунок (Slider)**                      | Ограничение, позволяющее линейное движение вдоль одной оси (поршень).                            |
| **Шаровой шарнир (Ball Socket)**           | Ограничение, позволяющее вращение во всех направлениях, но не линейное движение (плечо).         |
| **Фиксированное соединение (Fixed Joint)** | Жёсткое соединение двух тел, не позволяющее относительное движение.                              |
| **Пружина (Spring)**                       | Ограничение, создающее силу, пропорциональную растяжению (закон Гука).                           |
| **Мотор (Motor)**                          | Активный элемент ограничения, создающий движение (вращение шарнира, линейное движение ползунка). |
| **Лимиты (Limits)**                        | Ограничения на диапазон движения (минимальный/максимальный угол, расстояние).                    |
| **Жёсткость (Stiffness)**                  | Параметр, определяющий, насколько жёстко ограничение (высокая жёсткость = точное соблюдение).    |
| **Демпфирование (Damping)**                | Параметр, гасящий колебания в ограничении (трение, сопротивление).                               |

## Многопоточность и производительность

| Термин                                                | Объяснение                                                                            |
|-------------------------------------------------------|---------------------------------------------------------------------------------------|
| **JobSystem**                                         | Система задач JoltPhysics для параллельного выполнения физических вычислений.         |
| **Work stealing**                                     | Алгоритм балансировки нагрузки, когда простаивающие потоки "крадут" задачи у занятых. |
| **TempAllocator**                                     | Быстрый аллокатор с предварительным выделением памяти для временных данных симуляции. |
| **Кэширование форм (Shape Caching)**                  | Техника повторного использования форм для уменьшения аллокаций памяти.                |
| **Пространственное разбиение (Spatial Partitioning)** | Структуры данных для ускорения широкой фазы: grid, quad/octree, BVH.                  |
| **Сон тел (Sleeping)**                                | Оптимизация, при которой неактивные тела временно исключаются из симуляции.           |
| **Активация (Activation)**                            | Процесс "пробуждения" тела при воздействии силы или коллизии.                         |
| **Деактивация (Deactivation)**                        | Процесс перевода тела в спящий режим при отсутствии движения.                         |

## Интеграция с ProjectV

| Термин                             | Объяснение                                                                                                         |
|------------------------------------|--------------------------------------------------------------------------------------------------------------------|
| **HeightFieldShape**               | Специальная форма JoltPhysics для представления воксельного ландшафта. Эффективно использует разреженность данных. |
| **Voxel-based коллайдер**          | Коллайдер, построенный на основе воксельных данных. Может быть аппроксимирован HeightField или voxel grid.         |
| **ECS компоненты физики**          | Компоненты flecs для интеграции физики: `PhysicsBody`, `PhysicsProperties`, `PhysicsForces`.                       |
| **Синхронизация трансформаций**    | Процесс обновления трансформаций рендеринга (Vulkan) на основе результатов физической симуляции.                   |
| **Transform буфер**                | GPU буфер (Vulkan storage buffer) для передачи матриц трансформации из физики в рендерер.                          |
| **Object Layers для вокселей**     | Специализированные слои для воксельного мира: `VOXEL_TERRAIN`, `VOXEL_DYNAMIC`, `VOXEL_FLUID`.                     |
| **Детерминированная симуляция**    | Критично для воксельных миров, где изменения должны быть воспроизводимы.                                           |
| **Многопоточная обработка чанков** | Параллельная симуляция разных чанков воксельного мира через JobSystem.                                             |

## Специальные термины для воксельной физики

| Термин                                     | Объяснение                                                                                                             |
|--------------------------------------------|------------------------------------------------------------------------------------------------------------------------|
| **Воксельный коллайдер**                   | Коллайдер, представляющий воксельную геометрию. Может быть динамически обновляемым.                                    |
| **Сыпучие материалы (Granular Materials)** | Физика песка, гравия, сыпучих веществ. Моделируется через множество мелких частиц.                                     |
| **Жидкости (Fluids)**                      | Симуляция жидкостей в воксельном мире. Может использовать smoothed-particle hydrodynamics (SPH) или cellular automata. |
| **Деформация (Deformation)**               | Изменение формы воксельного ландшафта под воздействием сил (копание, взрывы).                                          |
| **Устойчивость (Stability)**               | Способность воксельных структур выдерживать нагрузки без разрушения.                                                   |
| **Разрушение (Fracture)**                  | Процесс разделения воксельной структуры на части при превышении предела прочности.                                     |
| **Пористость (Porosity)**                  | Свойство материала, определяющее наличие пустот. Влияет на физику жидкостей и прочность.                               |
| **Адгезия (Adhesion)**                     | Сила сцепления между вокселями разных материалов.                                                                      |

## Утилиты и преобразования

| Термин                     | Объяснение                                                                                                  |
|----------------------------|-------------------------------------------------------------------------------------------------------------|
| **JPH::Vec3**              | Вектор с тремя компонентами (float). Используется для позиций, сил, скоростей.                              |
| **JPH::Quat**              | Кватернион для представления вращения. Более стабилен, чем углы Эйлера.                                     |
| **JPH::Mat44**             | Матрица 4x4 для аффинных преобразований.                                                                    |
| **ToGlmVec3 / ToJoltVec3** | Функции преобразования между Jolt и glm векторами.                                                          |
| **ToGlmQuat / ToJoltQuat** | Функции преобразования между Jolt и glm кватернионами.                                                      |
| **Система координат**      | JoltPhysics использует правую СК с Y вверх. Vulkan использует правую СК с Y вниз. Требуется преобразование. |
| **Единицы измерения**      | JoltPhysics работает в метрической системе (метры, килограммы, секунды).                                    |

## Константы и лимиты

| Термин                            | Объяснение                                                  |
|-----------------------------------|-------------------------------------------------------------|
| **JPH::cMaxBodies**               | Максимальное количество тел в системе (по умолчанию 65536). |
| **JPH::cMaxBodyPairs**            | Максимальное количество пар тел для широкой фазы.           |
| **JPH::cMaxContactConstraints**   | Максимальное количество контактов за кадр.                  |
| **JPH::cCollisionSteps**          | Количество шагов коллизий за обновление (по умолчанию 1).   |
| **JPH::cMaxPhysicsJobs**          | Максимальное количество задач в JobSystem.                  |
| **JPH::cMaxPhysicsBarriers**      | Максимальное количество барьеров синхронизации.             |
| **JPH_VERSION_MAJOR/MINOR/PATCH** | Версия JoltPhysics (5.5.1 в ProjectV).                      |
