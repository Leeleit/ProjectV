# План улучшения документации ProjectV (v2.0)

**Статус:** Черновик для реализации  
**Дата:** 18 февраля 2026

## 1. Анализ текущих пробелов (Gaps Analysis)

### А. Модернизация C++ (C++20/23)

Текущие примеры тяготеют к стандарту C++17. Необходимо обновить документацию и примеры кода:

- **C++ Modules:** Перейти от `#include` к `import` в руководствах по сборке. Описать процесс модуляризации ядра движка
  для ускорения компиляции.
- **Concepts:** Заменить SFINAE и рантайм-проверки на концепты (например, в `GenericAudioSource` или ECS-компонентах).
- **Coroutines:** Внедрить `co_await` для асинхронной загрузки ассетов и генерации чанков (вместо колбэков в
  `resource-management.md`).
- **std::span:** Обновить API (особенно в `fastgltf` и `miniaudio`) для безопасной передачи массивов данных.

### Б. Глубокая работа с вокселями

- **Сжатие данных:** Добавить разделы по эффективному хранению (SVO, RLE, DAG) на стороне CPU до передачи в GPU-буферы.
- **Modification Pipeline:** Описать поток данных при изменении мира (Edit -> ECS Event -> Mesh Rebuild -> Physics
  Update -> NavMesh Update). Добавить диаграмму зависимостей.

### В. Сборка и зависимости

- **CMake Presets:** Описать использование пресетов для кросс-платформенной разработки (Windows/Linux). Создать
  отдельный файл в папке metadocs/cmake-guide, чтобы всё связанное с cmake было в этой папке, это логично.

### Г. Шейдерная инфраструктура

- **Hot-Reload:** Добавить описание системы мониторинга файлов шейдеров и их автоматической перекомпиляции.
- **Shader Languages:** Рассмотреть Slang или HLSL (через DXC) для улучшения модульности шейдеров.

## 2. Конкретные изменения в существующих документах

### 1. `docs/architecture/voxel-pipeline.md`

- **Task Shaders:** Добавить раздел про Mesh Shading pipeline (`VK_EXT_mesh_shader`). Описать использование Task Shaders
  для кластерного куллинга чанков.
- **Subgroup Operations:** Описать использование SIMD-инструкций на GPU для ускорения Greedy Meshing внутри Compute
  Shaders.

### 2. `docs/flecs/projectv-integration.md`

- **Task Graph:** Интегрировать `flecs.pipeline`. Показать, как использовать граф зависимостей вместо простого
  мультипоточного режима.
- **Query DSL:** Добавить примеры строковых запросов для динамических систем (необходимых для моддинга).

### 3. `docs/architecture/resource-management.md`

- **Smart Pointers vs Handles:** Добавить обоснование системы `ResourceHandle`. Сравнить с `std::shared_ptr` с
  кастомными делетерами для специфических задач (например, автоматическая выгрузка из GPU).
