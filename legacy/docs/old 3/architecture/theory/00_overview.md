## Архитектурные принципы

### Data-Oriented Design (DOD)

DOD фокусируется на организации данных для повышения производительности:

- Приоритет структур данных над алгоритмами
- Локальность данных (data locality) для минимизации промахов кэша
- Системы как чистые функции трансформации данных

### Entity Component System (ECS)

ECS реализует модульную архитектуру на основе композиции:

- **Компоненты:** только данные (POD-структуры)
- **Системы:** логика без внутреннего состояния
- **Сущности:** идентификаторы для ассоциации компонентов

## Пример: Воксельный чанк

```cpp
// DOD: Оптимизированная организация данных (SoA)
struct VoxelChunkData {
    std::vector<glm::vec3> positions;
    std::vector<uint32_t> colors;
    std::vector<uint8_t> materials;
};

// ECS: Компоненты
struct VoxelChunk {
    VoxelChunkData data;
    glm::ivec3 world_position;
};

struct NeedsMeshing {}; // Тег-компонент
struct IsVisible {};    // Тег-компонент

// Системы (функции трансформации)
void meshingSystem(VoxelChunk& chunk, const NeedsMeshing&) {
    // Генерация меша из данных вокселей
}
```

## Методы проектирования систем

1. **Анализ данных:** типы данных и частота обращений
2. **Организация памяти:** SoA vs AoS, hot/cold разделение
3. **Проектирование компонентов:** компактные POD-структуры
4. **Реализация систем:** функции трансформации данных
5. **Интеграция:** шина событий или синглтоны
