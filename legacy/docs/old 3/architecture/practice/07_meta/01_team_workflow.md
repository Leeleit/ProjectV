**🟡 Уровень 2: Средний** — Организация работы команды над воксельным движком.

## Обзор команды

| Роль                             | Ответственность                   | Технологический фокус    |
|----------------------------------|-----------------------------------|--------------------------|
| **Архитектор**                   | Общая архитектура, DOD/ECS дизайн | C++26, flecs, CMake      |
| **Графический программист**      | Vulkan рендерер, шейдеры          | Vulkan 1.4, GLSL, SPIR-V |
| **Физический программист**       | JoltPhysics, воксельная физика    | JoltPhysics, математика  |
| **Геймплей программист**         | Игровая логика, ECS системы       | flecs, SDL3 input        |
| **Инфраструктурный программист** | CI/CD, сборка, инструменты        | CMake, Python, Git       |
| **Технический художник**         | Контент, инструменты, pipeline    | MagicaVoxel, fastgltf    |

## Git Workflow

### Структура репозитория

```
ProjectV/
├── docs/                    # Документация (общая ответственность)
├── src/
│   ├── core/              # Ядро (Архитектор)
│   ├── graphics/          # Графика (Графический)
│   ├── physics/           # Физика (Физический)
│   ├── gameplay/          # Геймплей (Геймплей)
│   └── tools/             # Инструменты (Инфраструктурный)
├── external/              # Подмодули (Инфраструктурный)
└── scripts/               # Скрипты (Инфраструктурный)
```

### Ветвление (Git Flow адаптированный)

```
main ──────────────●──────────────────●───────────
                   ↑                   ↑
release/v0.1.0 ────●───────────────────●
                   ↑
develop ───────────●──●──●──●──●──●──●──●──●──●──
                   ↑     ↑     ↑
feat/graphics ─────●──●──●
                         ↑
feat/physics ────────────●──●──●
```

### Правила коммитов

```bash
# Формат: <тип>(<область>): <сообщение>

feat(graphics): добавлен базовый Vulkan рендерер
fix(physics): исправление коллизий вокселей
docs(workflow): обновление командного workflow
refactor(core): оптимизация ECS итераций
```

### Pull Request процесс

1. **Создание ветки** от `develop`
2. **Разработка** с регулярными коммитами
3. **Создание PR**: минимум 2 reviewers
4. **Code Review** (24-48 часов)
5. **Мёрж** после approval

## Спринт планирование

### Двухнедельные спринты

```yaml
Спринт #1 (Недели 1-2):
  Цель: Базовый Vulkan рендерер + SDL интеграция

  Задачи:
    - Архитектор: Дизайн ECS ядра (5 SP)
```

## Code Review

### Проверки для ProjectV

```cpp
// 1. DOD принципы:
// - Данные отделены от логики?
// - SoA вместо AoS где уместно?
// - Cache locality учтено?

// 2. Vulkan код:
// - Resource lifetime управляется правильно?
// - Нет утечек дескрипторов?
// - Validation layers включены в debug?

// 3. ECS код:
// - Компоненты — чистые данные?
// - Системы — чистые функции?
// - Легко тестировать?
```

## Инструменты

### Единая среда разработки

```yaml
Обязательные инструменты:
  - IDE: CLion 2023.3+ или VS Code
  - Compiler: MSVC 2022 (Windows) или GCC 13+ (Linux)
  - CMake: 3.25+
  - Git: 2.40+

Настройки CLion:
  - Форматирование: .clang-format
  - Inspection: .clang-tidy
```

### CI/CD Pipeline

```yaml
name: ProjectV CI
on: [push, pull_request]

jobs:
  build:
    strategy:
      matrix:
        os: [windows-latest, ubuntu-latest]
    runs-on: ${{ matrix.os }}
    steps:
    - uses: actions/checkout@v3
      with:
        submodules: recursive
    - run: cmake -B build
    - run: cmake --build build --parallel
```
