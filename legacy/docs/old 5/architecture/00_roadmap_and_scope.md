# Roadmap & Scope: ProjectV

**🔴 Уровень 3: Продвинутый** — Стратегический документ, определяющий границы проекта на разных этапах разработки.

---

## Введение

ProjectV — воксельный игровой движок, разрабатываемый в рамках проектной деятельности в вузе (ПГНИУ-МФТИ, 2-й курс).
Данный документ чётко разделяет **MVP (Minimum Viable Product)** для сдачи курсового проекта и **Vision** для будущей
инди-игры.

---

## MVP Scope: Курсовой проект

### Цель

Создать **фундамент воксельного движка**, демонстрирующий:

1. Владение современным C++ (C++23/26)
2. Понимание низкоуровневой графики (Vulkan 1.4)
3. Применение Data-Oriented Design и ECS
4. Интеграцию нескольких библиотек в единую архитектуру

### Обязательные компоненты

| Компонент               | Статус         | Описание                               |
|-------------------------|----------------|----------------------------------------|
| **Vulkan Renderer**     | ✅ Базовый      | Треугольник, текстуры, uniform buffers |
| **ECS Architecture**    | ✅ Flecs        | Сущности, компоненты, системы          |
| **Voxel Rendering**     | 🔄 В процессе  | 32³ чанк, Greedy Meshing               |
| **Physics Integration** | 🔄 JoltPhysics | Базовые коллизии, raycast              |
| **Destruction System**  | 📋 Планируется | Удаление/добавление вокселей           |

### Критерии успеха MVP

Для успешной сдачи проект должен демонстрировать:

| Критерий               | Метрика  | Цель                                   |
|------------------------|----------|----------------------------------------|
| **Производительность** | FPS      | ≥ 60 FPS при 1000+ видимых чанков      |
| **Стабильность**       | Crashes  | 0 критических ошибок при демо          |
| **Код**                | Стандарт | C++23, clang-tidy clean                |
| **Документация**       | Покрытие | Все ключевые системы задокументированы |
| **Демо**               | Сценарий | Полёт камерой, разрушение блоков       |

### Явно НЕ входит в MVP

Следующие функции **запрещены** для реализации в рамках MVP:

| ❌ Не делаем          | Причина                       |
|----------------------|-------------------------------|
| Мультиплеер          | Слишком сложно для курсового  |
| AI/Pathfinding       | Не относится к движку         |
| Сохранение мира      | Можно захардкодить демо-сцену |
| UI кроме debug       | ImGui достаточно              |
| Аудио                | Не критично для демо          |
| Кроссплатформенность | Windows-first для сдачи       |
| Оптимизация SVO      | Простые чанки достаточны      |

### Сроки MVP

| Этап       | Срок        | Демонстрация               |
|------------|-------------|----------------------------|
| Phase 1    | Март 2026   | Vulkan треугольник + Flecs |
| Phase 2    | Апрель 2026 | Воксельный чанк, камера    |
| Phase 3    | Май 2026    | Физика, разрушения         |
| **Защита** | Июнь 2026   | Полное демо                |

## Vision: Инди-игра

### Долгосрочная цель

Создать **симулятор-песочницу с масштабными боями**, объединяющий элементы:

| Игра                  | Что берём                       |
|-----------------------|---------------------------------|
| **War Thunder**       | Техника, физика разрушений      |
| **Foxhole**           | Масштабные бои, логистика       |
| **Hearts of Iron 4**  | Стратегический слой             |
| **Supreme Commander** | Стратегический zoom, юниты      |
| **Minecraft**         | Воксельный мир, строительство   |
| **SCP**               | Неевклидова геометрия, аномалии |

### Ключевые механики Vision

#### 1. Масштабные бои

- Тысячи юнитов на карте
- Техника от мотоциклов до танков
- Стратегический zoom (от солдата до карты)
- Динамический фронт

#### 2. Воксельное разрушение

- Полная разрушаемость среды
- Физика обломков (JoltPhysics)
- Процедурные кратеры от взрывов
- Строительство укреплений

#### 3. Симуляция мира

- Цепочки поставок (логистика)
- Добыча ресурсов
- Производство техники
- Динамическая экономика

#### 4. Неевклидова геометрия (SCP)

- Пространственные аномалии
- "Больше внутри, чем снаружи"
- Порталы без loading screens
- Нелинейные пространства

#### 5. Fluid Simulation

- Вода, лава, песок
- Cellular automata на GPU
- Взаимодействие с разрушениями

## Архитектурные решения для будущих фаз

### Phase 4: Networking (Post-MVP)

**Архитектура**: Server Authority + Client Prediction

| Решение              | Обоснование                                |
|----------------------|--------------------------------------------|
| Server Authority     | Физика и воксели контролируются сервером   |
| Client Prediction    | Плавное движение игрока без лагов          |
| Delta Compression    | Синхронизация только изменённых вокселей   |
| Spatial Partitioning | Разделение мира на зоны для load balancing |

**См.**: [Networking Concept](practice/21_networking-concept.md)

### Phase 5: SVO Rendering (Post-MVP)

**Архитектура**: Hybrid SVO + Chunks

| Решение            | Обоснование                             |
|--------------------|-----------------------------------------|
| SVO для LOD        | Compact storage, ray marching для теней |
| Chunks для близких | Точная геометрия, физика                |
| Streaming          | Потоковая загрузка далёких регионов     |

**См.**: [SVO Architecture](practice/00_svo-architecture.md)

### Phase 6: Fluid Simulation (Post-MVP)

**Архитектура**: GPU Cellular Automata

| Решение         | Обоснование                             |
|-----------------|-----------------------------------------|
| Compute Shaders | Параллельное обновление миллионов ячеек |
| Separate Grid   | Жидкости не требуют полного мира        |
| Interaction     | Воздействие на воксели (размывание)     |

## Защита от Scope Creep

### Правила добавления фич

1. **Вопрос**: Эта фича нужна для сдачи проекта?

- Да → Добавляем в MVP
- Нет → Переносим в Vision backlog

2. **Вопрос**: Эта фича увеличит сложность более чем на 20%?

- Да → Обязательно перенести в Vision
- Нет → Можно рассмотреть для MVP

3. **Вопрос**: Есть ли готовая библиотека?

- Да → Интеграция допустима
- Нет → Только если критично для MVP

### Red Flags (признаки scope creep)

- "Было бы круто добавить..."
- "Это займёт всего пару дней..."
- "Я уже начал писать..."
- "Давайте ещё и..."

**При обнаружении**: Остановиться, обновить документ, согласовать с командой.
