# Организация Памяти: Layout

Производительность ProjectV зависит от того, как процессор читает данные.
Современные процессоры **не читают память байтами**. Они читают **кэш-линиями (Cache Lines)** — блоками по 64 байта.

Цель: **Впихнуть как можно больше полезных данных в эти 64 байта.**

## 1. SoA (Structure of Arrays) vs AoS (Array of Structures)

### AoS (Array of Structures)

Традиционный подход ООП.

```cpp
struct Entity {
    vec3 pos;      // 12 байт
    float health;  // 4 байта
    char name[64]; // 64 байта
};
// В памяти: [pos][health][name]...[pos][health][name]
// Размер 1 объекта: 80 байт.
// Кэш-эффективность: Низкая.
```

Если системе нужно только `pos`, она всё равно загрузит `name` и `health` (мусор для этой системы).

### SoA (Structure of Arrays)

Подход DOD/ECS.

```cpp
vector<vec3> positions;
vector<float> healths;
// В памяти (positions): [pos][pos][pos][pos]...
// В кэш-линию (64 байта) влезает 5.3 вектора pos!
```

Если системе нужно только `pos`, она читает массив `positions`. Кэш заполнен на 100% полезными данными.
**Кэш-эффективность:** Высокая.

## 2. Выравнивание (Alignment) и Padding

Процессор любит читать данные по адресам, кратным их размеру (4, 8, 16, 64).
Компилятор добавляет "дырки" (Padding), чтобы выровнять поля.

### Плохо (Padding Hell)

```cpp
struct Bad {
    bool a;      // 1 байт
    double b;    // 8 байт (требует выравнивания на 8)
                 // -> +7 байт padding!
    int c;       // 4 байта
    float* d;    // 8 байт
                 // -> +4 байта padding!
};
// Размер: 32 байта. Полезных данных: 21 байт.
```

### Хорошо (Sorted by Size)

Сортируйте поля от больших к маленьким!

```cpp
struct Good {
    double b;    // 8 байт
    float* d;    // 8 байт
    int c;       // 4 байта
    bool a;      // 1 байт
    // -> +3 байта padding в конце (для выравнивания структуры)
};
// Размер: 24 байта. Экономия 25% памяти!
```

## 3. Воксельный Чанк: Практика

### Плохо (AoS)

```cpp
struct Voxel {
    vec3 pos;
    uint32_t color;
    uint8_t material;
    bool is_solid;
    // ... padding ...
};
// Размер вокселя ~32 байта.
// Чанк 32x32x32 = 32768 вокселей * 32 = 1 МБ.
```

### Хорошо (SoA + Bitpacking)

```cpp
struct VoxelChunkData {
    vector<vec3> positions;      // 12 байт
    vector<uint32_t> colors;     // 4 байта
    vector<uint8_t> materials;   // 1 байт
    vector<bool> solid_flags;    // 1 бит (std::vector<bool> упакован)
};
// Размер данных ~17 байт на воксель + отличная кэш-локальность.
```

В ProjectV мы используем **SoA** для массовых данных (воксели, частицы) и **AoS** для редких (игрок, камера), где
удобство важнее скорости.
