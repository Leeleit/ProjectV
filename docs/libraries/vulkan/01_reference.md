## Vulkan Overview

<!-- anchor: 00_overview -->

Низкоуровневый графический API для 3D-графики и вычислений.

**Vulkan** — кроссплатформенный API для высокопроизводительной 3D-графики, compute и ray tracing. Предоставляет тонкий
уровень абстракции над GPU, позволяющий полностью контролировать рендеринг и вычисления. В отличие от OpenGL, Vulkan
требует явного управления ресурсами, синхронизацией и памятью, что обеспечивает максимальную производительность за счёт
сложности API.

Основные возможности: GPU-driven rendering, async compute, ray tracing, mesh shading, bindless texturing.

---

## Требования

### Обязательные

- **C++11** или новее (рекомендуется C++17/C++20)
- **Vulkan SDK 1.3.250+** (рекомендуется последняя версия)
- **Совместимый GPU** с поддержкой Vulkan 1.2+
- **Драйверы GPU**: NVIDIA 470+, AMD 21.5.2+, Intel 27.20.100+

### Рекомендуемые инструменты

- **RenderDoc** — отладка GPU команд
- **Vulkan Validation Layers** — проверка корректности API вызовов
- **Nsight Graphics** (NVIDIA) или **Radeon GPU Profiler** (AMD) — профилирование

### Поддерживаемые платформы

- **Windows 10/11** (Win32, DirectX 12 через Vulkan)
- **Linux** (X11, Wayland) с NVIDIA/AMD драйверами
- **macOS** (MoltenVK) с ограничениями

## Ключевые особенности Vulkan

### Явное управление

В отличие от OpenGL/DirectX 11, Vulkan требует явного указания:

- **Синхронизации** — fences, semaphores, pipeline barriers
- **Памяти** — выделение, binding, management
- **Состояния** — pipelines, descriptor sets, render passes
- **Ресурсов** — создание, уничтожение, lifetime management

### Преимущества

| Аспект                   | Преимущество                                       |
|--------------------------|----------------------------------------------------|
| **Производительность**   | Минимальный driver overhead, прямое управление GPU |
| **Многопоточность**      | Параллельная запись command buffers                |
| **Контроль**             | Предсказуемое поведение, отсутствие "магии"        |
| **Кроссплатформенность** | Единый API для Windows, Linux, Android             |

### Недостатки

| Аспект         | Недостаток                                    |
|----------------|-----------------------------------------------|
| **Сложность**  | Высокий порог входа, много boilerplate        |
| **Объём кода** | ~1000 строк для треугольника                  |
| **Отладка**    | Сложнее находить ошибки без validation layers |
