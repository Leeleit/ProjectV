## Zstandard (zstd) — Обзор

<!-- anchor: 00_overview -->


## Что такое Zstandard

Zstandard (zstd) — алгоритм сжатия данных без потерь, разработанный Яном Колле (Yann Collet) в Facebook в 2015 году.
Цель проекта — обеспечить высокую скорость сжатия и распаковки при сохранении хорошего коэффициента сжатия.

## Ключевые характеристики

| Характеристика      | Значение                   |
|---------------------|----------------------------|
| Скорость сжатия     | 300–500 MB/s (уровень 3)   |
| Скорость распаковки | 800–1200 MB/s              |
| Уровни сжатия       | от -131072 до 22           |
| Отношение сжатия    | сопоставимо с zlib/deflate |
| Лицензия            | BSD-3-Clause               |

## Основные возможности

### Гибкие уровни сжатия

Zstd поддерживает широкий диапазон уровней сжатия:

- **Отрицательные уровни** (−1 до −131072): максимальная скорость, минимальное сжатие
- **Положительные уровни** (1–22): баланс скорости и сжатия
- **Уровень по умолчанию**: 3

Важное свойство: скорость распаковки не зависит от уровня сжатия. Файл, сжатый с уровнем 22, распаковывается так же
быстро, как и сжатый с уровнем 1.

### Словарное сжатие

Для небольших данных (< 64 KB) zstd поддерживает предварительное обучение словаря на репрезентативных образцах. Это
значительно улучшает степень сжатия для однотипных данных.

### Потоковый API

Zstd поддерживает обработку данных потоком, что позволяет работать с данными произвольного размера без загрузки в память
целиком.

### Мультифреймовая структура

Несколько сжатых фреймов можно конкатенировать в один файл. Декомпрессор автоматически распознает и обработает все
фреймы последовательно.

## Сравнение с другими алгоритмами

| Алгоритм     | Скорость сжатия | Скорость распаковки | Степень сжатия |
|--------------|-----------------|---------------------|----------------|
| **zstd**     | 300–500 MB/s    | 800–1200 MB/s       | Высокая        |
| LZ4          | 700–1000 MB/s   | 2000–4000 MB/s      | Средняя        |
| zlib/deflate | 20–50 MB/s      | 100–200 MB/s        | Высокая        |
| LZMA/xz      | 1–5 MB/s        | 30–50 MB/s          | Очень высокая  |

## Применение

Zstd подходит для:

- Сжатия данных в реальном времени (сетевые протоколы, логи)
- Хранения структурированных данных (базы данных, файлы конфигурации)
- Архивации файлов
- Сжатия в играх (ресурсы, сохранения, сетевой трафик)
- Компрессии больших объёмов данных (backup, логи)

## Требования

- **C99** или новее для C API
- **C++11** или новее для C++ обёрток
- **CMake 3.10+** для сборки

## Поддерживаемые платформы

- Windows (x86, x64, ARM, ARM64)
- Linux (x86, x64, ARM, ARM64, RISC-V)
- macOS (x64, ARM64)
- FreeBSD, OpenBSD
- Android, iOS
